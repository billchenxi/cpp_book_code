# Chapter 10 Code

This folder groups the code referenced by chapter 10 of `B22398_10.docx`.

Files in this chapter:

- `tinynet.h`: the TinyNet CNN used across the chapter.
- `device_utils.h`: small helper for CUDA availability checks in the chapter 10 LibTorch demos.
- `trace_and_save_ts.cpp`: export TinyNet to TorchScript from C++.
- `infer_torchscript.cpp`: load and run a TorchScript artifact.
- `parity_check.cpp`: compare native TinyNet output with the TorchScript artifact.
- `onnx_loader_demo.cpp`: load and run a compatible ONNX model with ONNX Runtime.
- `micro_batcher_demo.cpp`: bounded micro-batching scheduler with LibTorch inference.
- `fp16_torchscript_demo.cpp`: FP16 TorchScript inference on CUDA.
- `onnx_int8_demo.cpp`: load and run a quantized ONNX graph.
- `pruned_torchscript_demo.cpp`: load a pruned TorchScript artifact.
- `benchmark_student_vs_baseline.cpp`: compare throughput for two TorchScript models.
- `cuda_graph_capture_demo.cpp`: capture and replay a CUDA graph for fixed-shape inference.
- `onnx_session_options_demo.cpp`: minimal ONNX Runtime session-options snippet as a runnable file.

## Start

All commands below assume you run them from the repository root.

LibTorch checks:

```bash
clang++ --version
ls ./libtorch/include/torch/torch.h
```

ONNX Runtime checks:

```bash
brew --prefix onnxruntime
ls "$(brew --prefix onnxruntime)/include/onnxruntime/onnxruntime_cxx_api.h"
```

CUDA-only checks for the GPU-specific examples:

```bash
nvcc --version
nvidia-smi
```

## TorchScript path

Export TinyNet to TorchScript:

```bash
g++ -std=c++17 chapter_10/trace_and_save_ts.cpp -o chapter_10/trace_and_save_ts \
  -I ./libtorch/include \
  -I ./libtorch/include/torch/csrc/api/include \
  -L ./libtorch/lib \
  -Wl,-rpath,./libtorch/lib \
  -ltorch -ltorch_cpu -lc10

DYLD_LIBRARY_PATH=./libtorch/lib ./chapter_10/trace_and_save_ts
```

Load and run the exported artifact:

```bash
g++ -std=c++17 chapter_10/infer_torchscript.cpp -o chapter_10/infer_torchscript \
  -I ./libtorch/include \
  -I ./libtorch/include/torch/csrc/api/include \
  -L ./libtorch/lib \
  -Wl,-rpath,./libtorch/lib \
  -ltorch -ltorch_cpu -lc10

DYLD_LIBRARY_PATH=./libtorch/lib ./chapter_10/infer_torchscript chapter_10/tinynet.ts
```

Run the native-vs-TorchScript parity check:

```bash
g++ -std=c++17 chapter_10/parity_check.cpp -o chapter_10/parity_check \
  -I ./libtorch/include \
  -I ./libtorch/include/torch/csrc/api/include \
  -L ./libtorch/lib \
  -Wl,-rpath,./libtorch/lib \
  -ltorch -ltorch_cpu -lc10

DYLD_LIBRARY_PATH=./libtorch/lib ./chapter_10/parity_check chapter_10/tinynet.ts
```

Run the micro-batcher demo:

```bash
g++ -std=c++17 -pthread chapter_10/micro_batcher_demo.cpp -o chapter_10/micro_batcher_demo \
  -I ./libtorch/include \
  -I ./libtorch/include/torch/csrc/api/include \
  -L ./libtorch/lib \
  -Wl,-rpath,./libtorch/lib \
  -ltorch -ltorch_cpu -lc10

DYLD_LIBRARY_PATH=./libtorch/lib ./chapter_10/micro_batcher_demo chapter_10/tinynet.ts
```

## ONNX Runtime path

The ONNX examples expect artifacts that are not generated by the C++ code in this folder:

- `chapter_10/tinynet.onnx` for `onnx_loader_demo.cpp`
- `chapter_10/model.int8.onnx` for `onnx_int8_demo.cpp`

The chapter text explicitly notes that ONNX export is usually handled in Python or CI. Once you have a compatible ONNX model, build and run these files as follows.

Load a standard ONNX model:

```bash
g++ -std=c++17 -O2 chapter_10/onnx_loader_demo.cpp -o chapter_10/onnx_loader_demo \
  -I"$(brew --prefix onnxruntime)/include" \
  -L"$(brew --prefix onnxruntime)/lib" \
  -Wl,-rpath,"$(brew --prefix onnxruntime)/lib" \
  -lonnxruntime

./chapter_10/onnx_loader_demo chapter_10/tinynet.onnx
```

Load a quantized ONNX model:

```bash
g++ -std=c++17 -O2 chapter_10/onnx_int8_demo.cpp -o chapter_10/onnx_int8_demo \
  -I"$(brew --prefix onnxruntime)/include" \
  -L"$(brew --prefix onnxruntime)/lib" \
  -Wl,-rpath,"$(brew --prefix onnxruntime)/lib" \
  -lonnxruntime

./chapter_10/onnx_int8_demo chapter_10/model.int8.onnx
```

Run the session-options snippet:

```bash
g++ -std=c++17 -O2 chapter_10/onnx_session_options_demo.cpp -o chapter_10/onnx_session_options_demo \
  -I"$(brew --prefix onnxruntime)/include" \
  -L"$(brew --prefix onnxruntime)/lib" \
  -Wl,-rpath,"$(brew --prefix onnxruntime)/lib" \
  -lonnxruntime

./chapter_10/onnx_session_options_demo
```

## Notes

- `tinynet.h` was moved from the old `ch10/` folder into `chapter_10/`.
- `trace_and_save_ts.cpp` and `parity_check.cpp` both fix the random seed so the parity check matches the traced TinyNet artifact generated by this chapter.
- `fp16_torchscript_demo.cpp` and `cuda_graph_capture_demo.cpp` are meaningful only on a CUDA-capable machine.
- `cuda_graph_capture_demo.cpp` also requires a CUDA-enabled LibTorch build, not only a CUDA runtime.
- `pruned_torchscript_demo.cpp`, `benchmark_student_vs_baseline.cpp`, `onnx_loader_demo.cpp`, and `onnx_int8_demo.cpp` are loaders and harnesses. They require compatible model artifacts produced elsewhere.
- If an ONNX Runtime executable fails at launch with a missing `libprotobuf...dylib`, reinstall or relink `protobuf` and `onnxruntime` so their Homebrew dylib versions match.
